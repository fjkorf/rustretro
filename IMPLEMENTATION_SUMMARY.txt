PROJECT OVERVIEW
================
RustRetro is a lightweight, functional libretro frontend written in Rust.
It demonstrates proper FFI interaction with libretro cores, SDL2 integration,
and the callback system required for emulation frontends.

COMMAND-LINE INTERFACE
======================
Usage: rustretro [OPTIONS] --core <PATH> --rom <PATH>

Required Arguments:
  --core <PATH>        Path to libretro core (.so/.dll/.dylib)
  --rom <PATH>         Path to ROM/content file

Optional Arguments:
  --scale <FACTOR>     Window scale factor [default: 3]
  --save-dir <PATH>    Save file directory [default: .]
  --system-dir <PATH>  BIOS directory [default: .]
  --fullscreen         Enable fullscreen mode
  --no-audio           Disable audio output
  --help               Show help message

Example Usage:
  cargo run --release -- \
    --core ./snes9x_libretro.so \
    --rom ./mario.sfc \
    --scale 2

ARCHITECTURE HIGHLIGHTS
=======================
1. Modular Design
   - main.rs:      Pure CLI handler, no core logic
   - libretro.rs:  Core loading and FFI, reusable
   - sdl_interface.rs: SDL abstractions, replaceable
   - frontend.rs:  Main logic, testable callbacks

2. Safe FFI
   - Minimal unsafe code (only at boundaries)
   - Safe string conversions from C
   - Static callback dispatch pattern
   - Atomic pointer for callback context

3. Callback System
   - Uses atomic static pointer to context
   - Static C-compatible functions
   - Allows C code to call Rust methods
   - Thread-safe with SeqCst ordering

4. Error Handling
   - Custom LibretroError enum with thiserror
   - Detailed error messages
   - Result-based error propagation
   - Validation of inputs at CLI level

HOW TO USE
==========
1. Build:
   cd /path/to/rustretro
   cargo build --release

2. Obtain libretro cores from:
   https://buildbot.libretro.com/nightly/

3. Run:
   ./target/release/rustretro \
     --core /path/to/core.so \
     --rom /path/to/game.rom

4. Control with keyboard (see INPUT MAPPING above)

5. Press ESC or close window to exit
